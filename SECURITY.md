# Security Policy

## Supported Versions

We take the security of this project seriously. Currently, we provide security updates and patches for the following major versions:

| Version | Supported          | Notes |
| ------- | ------------------ | ----- |
| 3.x.x   | :white_check_mark: | Active development and all security updates. |
| 2.x.x   | :white_check_mark: | Critical security patches only. |
| < 2.0   | :x:                | End of life. No longer receiving updates. |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please report it by creating a new issue on our GitHub repository.

1. **Create an Issue:** Navigate to the [Issues](../../issues) tab and click "New Issue".
2. **Provide Details:** Include a clear and detailed description of the vulnerability. Be sure to provide the exact steps to reproduce the issue, the expected behavior, and the potential impact.
3. **Labeling:** If possible, please tag the issue with a `security` or `bug` label so it can be quickly identified and triaged.

### What to expect

* **Triage:** We monitor new issues closely and will review your report to confirm whether the vulnerability is valid.
* **Resolution:** If the vulnerability is accepted, we will prioritize developing a patch and will keep the issue updated with our progress.
* **Disclosure:** Once the patch is released, we will publicly acknowledge your contribution in our release notes and close the issue.
* **Declined Reports:** If the report is declined (e.g., it is intended behavior or out of scope), we will clearly explain our reasoning before closing the issue.

Thank you for helping keep this project and its users safe!
